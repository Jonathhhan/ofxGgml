# clang-tidy

`ofxGgml` ships with a repo-level `.clang-tidy` configuration and helper scripts for running static analysis against the addon and the GUI example without changing the normal openFrameworks build flow.

## What is included

- repo-wide `.clang-tidy`
- `scripts/run-clang-tidy.ps1` for Windows / Visual Studio workflows
- `scripts/run-clang-tidy.sh` for Linux/macOS and compile-database workflows

The default configuration is intentionally practical rather than maximal:

- it targets addon and example sources under `src/` and `ofxGgmlGuiExample/src/`
- it enables `bugprone`, `performance`, `portability`, `readability`, and `modernize` checks
- it avoids noisy rules that do not fit openFrameworks-heavy codebases well

## Recommended workflows

### Windows / Visual Studio

Use the PowerShell helper:

```powershell
./scripts/run-clang-tidy.ps1
./scripts/run-clang-tidy.ps1 -Project ofxGgmlGuiExample/ofxGgmlGuiExample.vcxproj
./scripts/run-clang-tidy.ps1 -UseMsBuild
./scripts/run-clang-tidy.ps1 -Files src/core/ofxGgmlCore.cpp,src/inference/ofxGgmlInference.cpp
```

The script prefers a `compile_commands.json` database when one is available. If no compile database is found and `-UseMsBuild` is supplied, it falls back to Visual Studio's Clang-Tidy integration for the given `.vcxproj`.

### Linux / macOS

Use the shell helper:

```bash
./scripts/run-clang-tidy.sh
./scripts/run-clang-tidy.sh src/core/ofxGgmlCore.cpp src/inference/ofxGgmlInference.cpp
./scripts/run-clang-tidy.sh --fix
```

This path expects a usable `compile_commands.json`, typically generated by a CMake or compatible build directory.

## Compile database notes

The helper scripts look for `compile_commands.json` in common locations:

- `./compile_commands.json`
- `./build/compile_commands.json`
- `./ofxGgmlGuiExample/compile_commands.json`
- `./ofxGgmlGuiExample/build/compile_commands.json`

If your build exports the compile database somewhere else, pass it explicitly:

```powershell
./scripts/run-clang-tidy.ps1 -BuildPath path/to/build
```

```bash
./scripts/run-clang-tidy.sh --build-path path/to/build
```

## Scope

The helper scripts are intended for:

- addon code in `src/`
- example code in `ofxGgmlGuiExample/src/`

They are not meant to lint vendored third-party code under `libs/`.

## Tips

- Start with a few files before running over the whole tree.
- Prefer fixing `bugprone` and `performance` warnings first.
- Keep `clang-tidy` advisory unless your team has already normalized the warning set.
